# work-report skill 設計

## 背景・目的

社内向けの作業内容報告が月・水・金の週3回ある。毎回「先週〜今週何やったっけ」を思い出すのが手間なので、対象期間の作業内容を自動集計し、報告にそのまま使える箇条書き形式でチャット上に出力するskillを作る。

既存の `daily-review`（日次の振り返り・メモリ整理）や `weekly-report`（KPT・生産性指標込みの正式な週報ドキュメント生成）とは目的が異なる。本skillはドキュメントとして残す想定がなく、コピペして報告するための使い捨てメモを素早く作ることが目的。

## スコープ外

- weekly-report が持つ KPT・生産性指標（就寝/起床/調子）・ルーティン達成率の集計
- ファイルへの保存・git commit・push
- 週報のような「サマリー文章化」（要約・統合はするが、KPTや所感などの追加セクションは作らない）

## 対象期間の決定ロジック

実行日の曜日によって起点が変わる。

| 実行曜日 | 対象期間 |
|---|---|
| 金曜 | 今週月曜 〜 今日 |
| 上記以外（月・水・その他任意の曜日） | 先週の同じ曜日 〜 今日 |

例:
- 月曜実行 → 先週月曜 〜 今日
- 水曜実行 → 先週水曜 〜 今日
- 金曜実行 → 今週月曜 〜 今日（先週金曜の作業は前回報告済みのため含めない）

土日も対象期間に含める場合はデータ取得を試みるが、作業実績がないのが正常系のため、0件でもエラー扱いにしない。

```bash
today=$(TZ=Asia/Tokyo date +%Y-%m-%d)
dow=$(TZ=Asia/Tokyo date +%u)  # 1=Mon ... 5=Fri, 7=Sun

if [ "$dow" -eq 5 ]; then
  # 金曜: 今週月曜起点
  start_date=$(TZ=Asia/Tokyo date -d "$today -$((dow-1)) days" +%Y-%m-%d)
else
  # それ以外: 先週の同じ曜日起点
  start_date=$(TZ=Asia/Tokyo date -d "$today -7 days" +%Y-%m-%d)
fi
end_date="$today"
```

## データソース

`daily-review` のデータ取得ロジックを踏襲し、対象期間分をまとめて取得する。個人リポジトリ（`~/.dotfiles`、`aya-215/life` など）は報告対象外のため取得しない。

| ソース | 取得先 | 用途 |
|---|---|---|
| 日報の📝サマリー（Work欄） | `aya-215/life` の日報 Issue（open）または `blog/YYYYMMDD.md`（close済み） | 対象期間内の各日から Work セクションを抽出 |
| gitログ（AIカテゴリ） | `~/src/github.com/ebase-dev/*`、`/mnt/d/tomcat/webapps/eb-api-extended` | コミットログから作業内容を補完（日報に書き漏れがあっても拾える） |
| gitログ（その他カテゴリ） | `/mnt/d/tomcat/webapps/hankyu`（社内GitBucketリポジトリ） | 同上 |
| Rocket Chat（`mori.a-times`） | `mcp__rocketchat__*` ツール | 自分の作業・レビュー作業を対象期間で取得 |
| Claudeセッション要約 | `~/.nb/claude/sessions/YYYY-MM-DD/*.md` | gitログで拾えない「その他」の仕事関連作業（`hankyu`以外）を補完 |

日報Issue特定・close済みblog参照・gitログ取得・Rocket Chat取得のコマンド例は `daily-review` skill の実装をそのまま流用する。

## 分類ロジック

出力は「カテゴリ → サブプロジェクト（=リポジトリ名 or 作業テーマ）→ 箇条書き」の2階層構造。個人リポジトリの作業はいずれのカテゴリにも含めない。

### カテゴリ判定

| カテゴリ | 対象 |
|---|---|
| AI | `ebase-dev` 配下の全リポジトリ ＋ `eb-api-extended` |
| その他 | 仕事関連かつAIカテゴリに属さないもの（`hankyu` のgitログ、および日報・Claudeセッション要約から判明したAI以外の仕事関連作業） |

### サブプロジェクト名

原則リポジトリ名をそのまま使う。例外は以下の固定マッピングのみ:

| リポジトリ名 | 表示名 |
|---|---|
| `ebase-portal-chat` | `AI-search` |
| `hankyu` | 阪急キッチンエール |

新規の例外が発生した場合は、このマッピング表をskill内に手動追記する運用とする（自動推測は行わない）。gitログに紐づかない作業（日報・Claudeセッション要約のみから判明した作業）は、内容から推測できる作業テーマ名をサブプロジェクト名とする。

## 出力形式

コピペしやすいプレーンテキストの箇条書き。ファイル保存はせず、チャット上にそのまま表示する。

```
- AI
    - MCP
        - 新規リソースの作成
    - AI-search
        - takashoブランドの作成
        - 検証環境の調整
        - 詳細画面URL対応
- その他
    - 阪急キッチンエール
        - 締め日マスタボタンの作成
```

- 各項目はタイトルのみ（説明文・PR番号などの付加情報なし）、名詞句で統一
- 日報サマリー・gitログ・Rocket Chat・Claudeセッション要約から集めた内容のうち、同一作業を指すものは統合して重複を排除
- 活動がないカテゴリ・サブプロジェクトは出力しない
- 個人開発の話題（dotfiles等）が日報やClaudeセッション要約に含まれていても出力に含めない
- KPTや所感、指標集計などの追加セクションは作らない

## エラー処理

| 状況 | 対応 |
|---|---|
| 対象期間内の日報が1日も見つからない | gitログ・Rocket Chat・Claudeセッション要約のみで生成を続行 |
| gitリポジトリなし | スキップして他のソースで生成 |
| Rocket Chat発言が0件 | スキップして他のソースで生成 |
| Claudeセッション要約なし（`sessions/YYYY-MM-DD/` が空） | スキップして他のソースで生成 |
| 入力ソースがすべて空 | 「対象期間の作業内容が見つかりません」と表示して終了 |
| 土日にデータがない | 正常系として扱い、警告等は出さない |
