# 外部アクション承認ルール

## 原則

リポジトリ外・自分のマシン外に影響が出る操作（外部アクション）は、
**ユーザーの明示的な承認なしに実行しない**こと。

外部アクションの例:

- PRコメント・レビューコメントの投稿（`gh pr comment` / `gh pr review` / `mcp__gitbucket__gitbucket_add_comment`）
- Issue・PRの作成・更新・クローズ
- Rocket Chat等へのメッセージ送信
- Artifactの公開、メール送信

## 「確認して」「レビューして」の解釈

「確認・レビュー・検証して」という依頼は**分析と報告のみ**を意味する。
分析結果を提示して止まり、投稿・修正等のアクションは承認を待つこと。

## 実装・テストの開始も同様

調査・説明を頼まれた段階では分析で止まる。
テスト作成やコード実装は、計画にユーザーが明示的にOKを出してから始めること。

例外: ユーザーがそのターンで明示的に依頼した操作（「PRにこのコメントを投稿して」等）は承認済みとみなす。

補足: `ask` permission（settings.json）でも同系統の操作に確認ダイアログを出す決定的ガードを併用している。
